# EVENT_ASSET_PLATFORM_SPEC.md

## Product Vision
Build a QR-first event and asset booking platform where users can discover events, host events, and book assets like mobile fridges and toilets.

---

## Core Problem
- Manual booking processes
- No visibility of availability
- No centralized event discovery
- No viral growth loop

---

## System Overview
Core modules:
- User System
- Asset Management
- Booking Engine
- Event System
- QR System
- Notification System
- Event Discovery Hub

---

## User Roles
- Asset Owner
- Event Organizer
- Attendee

---

## Core User Flows

### QR to Booking
1. Scan QR
2. View availability
3. Select slot
4. Submit booking
5. Email notification
6. Owner confirms

### Event Creation
1. Create event
2. Set public/private
3. Generate QR

### Event Discovery
1. Browse events
2. View details
3. RSVP

---

## Core Features
- Booking calendar
- QR code system
- Event creation
- Asset management
- Notifications

---

## Business Model
Phase 1: Free
Phase 2: Commission per booking
Phase 3: Subscription model

---

## Technical Stack
- Frontend: React
- Backend: .NET or Node.js
- Database: PostgreSQL/MySQL
- Hosting: Vercel + API backend

---

## MVP Scope (Phase 1)
- User registration
- Asset listing
- Booking calendar
- Email notifications
- QR booking access

---

## Phase Rollout

### Phase 1
- Booking system
- QR integration

### Phase 2
- Event system

### Phase 3
- Discovery + social

### Phase 4
- Payments + scale

---

## Success Metrics
- Bookings
- QR scans
- Event creation
- Repeat usage

---

## Risks
- Overbuilding
- No demand
- Complexity creep

---

## Key Principle
Focus on bookings first. Everything else comes later.
