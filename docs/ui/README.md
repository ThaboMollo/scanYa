# UI Implementation Docs

This folder contains the frontend redesign plans for the Direction C product structure:

- Public marketplace experience
- Logged-in operational workspace

## Recommended UI Build Order
1. Route split and layout shells
2. Design tokens and reusable primitives
3. Homepage redesign
4. Asset browse and asset detail redesign
5. Workspace shell and owner dashboard
6. Owner asset management redesign
7. Booking management redesign
8. QR and event-ready surfaces

## Core UI Principle
Do not present discovery, booking, and owner operations in the same screen. Public users and workspace users should feel like they are in two connected but purpose-built products.
